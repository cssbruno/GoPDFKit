## Strongly Recommended

| Item | Status | Notes |
| --- | --- | --- |
| `CompressionPolicy` | Done | Construction and output-time support exist. |
| Explicit `FontCache` options | Done | `WithFontCache`, `WithUTF8FontCache`. |
| `OutputOptions` | Done | Covers sync, compression, limits, deterministic mode. |
| `Limits` | Mostly done | Attachment, image source/decoded, HTML input/generated pages, page count, imported PDF bytes, and imported referenced-object limits are enforced. Template serialization is still global rather than per-document. |
| `SecurityPolicy` | Partial | JavaScript, local HTML images, file-backed attachments, raw writes, and legacy protection are gated. Broader signing/import policy should be considered separately. |
| Template interface split | Planned | Keep compatibility surface while moving internals to narrower interfaces. |
| Fuzz targets | Done | Initial fuzz targets cover HTML, SVG, templates, images, imported PDFs, and CMS/DER parsing. |
| Context cancellation | Mostly done | Output boundary APIs, HTML writing, and page-compression workers honor context; parser-level cancellation remains future work. |
| Observability hooks | Partial | Attachment load, page compression, output object, and document image cache hooks exist; font/import cache hit/miss hooks still need integration. |

## Deferred

| Item | Status | Notes |
| --- | --- | --- |
| Full loader abstraction | Deferred | Prefer narrow loaders: image, attachment, UTF-8 font, imported PDF. |
| Benchmark automation | Deferred | Manual budgets are documented in `benchmark-budgets-v0.9.md`. |
| AES-based modern PDF encryption | Deferred | Out of scope for production-stability release. |
